## Purpose

Normalize Equipment business workflow boundaries for equipment management and checkout/return ownership.

This milestone keeps direct equipment CRUD/status endpoints HTTP-only at the controller boundary and moves organization context enforcement into equipment policy/service.

It also records the current checkout/return workflow reality: operational checkout/return is represented through Field Session scan workflow and equipment/session movement records, not by arbitrary manual equipment status mutation.

## Business Workflow Before

`equipment.controller.js` derived organization context and asserted organization presence before calling the equipment service.

`equipment.service.js` owned equipment creation, update, status update, and deletion workflow, but organization context enforcement was not centralized in policy.

The Field Session service currently owns scan-based operational movement behavior for prepared/scanned/returned item states and creates equipment movement records inside a transaction.

## Business Workflow After

`equipment.policy.js` now exposes:

```text
assertEquipmentOrganizationContext
assertEquipmentExists
assertManualStatusUpdateAllowed
```

`equipment.service.js` now asserts organization context before every organization-scoped equipment workflow:

- list equipment
- get equipment by id
- create equipment
- update equipment
- update equipment status
- delete equipment
- find or create runtime category by code

`equipment.controller.js` now remains HTTP orchestration only:

- reads request context, params, and body
- calls equipment service
- maps response DTOs
- returns success response
- forwards errors

## Checkout / Return Ownership

Direct manual equipment status mutation is intentionally constrained by `assertManualStatusUpdateAllowed`.

Operational checkout/return belongs to the Field Session scan workflow, where scan mode decides the next session item status and creates an equipment movement record inside the field session transaction.

Current ownership:

```text
Equipment service
  owns equipment master/runtime equipment CRUD and guarded manual status updates

Field Session service
  owns checkout/return-style scan orchestration, session item status transition, and equipment movement creation

Equipment repository
  owns equipment data access only

Field Session repository
  owns session item and movement data access only
```

This means BE-05.06 does not introduce a new standalone checkout/return endpoint. It documents the existing workflow owner and keeps equipment manual status updates from bypassing required lifecycle workflow.
