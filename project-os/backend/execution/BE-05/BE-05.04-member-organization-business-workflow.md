## Business Workflow Before

Member and Organization controllers already delegated to service methods and stayed mostly HTTP-focused.

The services owned workflow orchestration, but context checks for organization/user identity were implemented as service-local guards that directly threw context errors.

## Business Workflow After

Member policy now exposes:

```text
assertMemberOrganizationContext
assertMemberUserContext
assertMemberExists
```

Organization policy now exposes:

```text
assertOrganizationContext
assertOrganizationExists
```

The services still own workflow orchestration but delegate reusable context boundary assertions to policy.

## API Contract

No intentional API contract change.

Controller exports, service method names, response mapper use, HTTP status behavior, and repository method calls were preserved.

## Validation Boundary

## Policy Boundary

Policy files now own context guard semantics for:

- member organization context;
- member user context;
- organization context;
- member existence;
- organization existence.

This aligns with BE-05 service orchestration and prepares future BE-07 policy/domain rule work without changing current API behavior.
