## Legacy Cleanup

The following legacy validator was removed under the BE-06 Early Development Refactor Policy:

```text
fieldops-be/src/modules/auth/auth.validators.js
```

Reason:

```text
Auth validation had a Joi-based local validateBody helper and used a non-standard AppError path.
Routes were migrated to auth.validation.js using the shared core validation foundation.
The legacy file had no remaining route references after migration.
```

## Review Result

Review confirms:

```text
route-level validation now exists for target mutation endpoints
params validation was added where route params are consumed by controllers
body validation uses the shared core validation foundation in normalized modules
Auth no longer uses the legacy Joi validator file
Equipment and Field Session no longer use module-local validateBody helpers
request validation errors flow through REQUEST_VALIDATION_ERROR
validators do not import Prisma
repository files were not modified
Prisma schema and migrations were not modified
frontend files were not modified
```
