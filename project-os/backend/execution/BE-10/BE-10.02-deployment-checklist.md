# BE-10.02 — Deployment Checklist

Status: Standard
Scope: Backend Production Readiness
Owner: Backend Architecture
Parent Package: `BE-10-production-readiness.md`
Previous Milestone: `BE-10.01-production-checklist.md`

## Purpose

BE-10.02 defines the backend deployment checklist for a target release scope.

The checklist ensures that deployment inputs, configuration expectations, migration risk, operational ownership, rollout sequence, and post-deploy verification are known before production delivery.

## Release Scope

Use this section to name the exact backend release slice being deployed.

```text
Release scope: required for completed review
Target modules: required for completed review
Target environment: required or NOT_APPLICABLE with reason
Deployment owner: required or NOT_APPLICABLE with reason
Related commits: required for completed review
Related PR: required when available
```

## Preconditions

Before BE-10.02 can pass, the target release scope must have:

```text
□ BE-10.01 Production Checklist completed or explicitly deferred
□ changed backend files identified
□ runtime entrypoint impact reviewed
□ environment/configuration impact reviewed
□ database migration impact reviewed when applicable
□ rollback owner identified when needed
□ post-deploy smoke check owner identified
```

## Deployment Inputs

```text
□ commit list is known
□ deploy branch or release tag is known
□ target environment is known
□ required secrets or config variables are known
□ database migration requirement is known
□ runtime command or service restart requirement is known
□ external dependency impact is known
```

## Configuration Checklist

```text
□ no `.env` file contents are committed
□ required environment variables are documented when changed
□ default behavior is safe when optional configuration is absent
□ production-only settings are identified
□ staging or preview verification path is known when applicable
```

## Migration Risk Checklist

```text
□ no schema migration is included, or migration owner is identified
□ destructive migration risk is reviewed
□ backward compatibility is reviewed when old and new code may overlap
□ seed or data repair steps are documented when needed
□ migration rollback or mitigation is known when needed
```

## Rollout Checklist

```text
□ deployment order is known
□ service restart or redeploy requirement is known
□ queue/outbox/background worker impact is reviewed when applicable
□ expected temporary degraded states are documented
□ responsible human or steward is identified for deploy observation
```

## Post-Deploy Verification Checklist

```text
□ health check or equivalent runtime signal is known
□ smoke check path is known
□ critical endpoint or workflow is identified
□ logs can be inspected with safe context
□ failure signal and escalation path are known
```

## Deployment Result

Use one result only.

```text
DEPLOYMENT_READY
DEPLOYMENT_READY_WITH_NOTES
DEPLOYMENT_NOT_READY
DEPLOYMENT_NOT_REQUIRED
```

## Known Exceptions

Record all exceptions clearly. Do not leave placeholders in a completed review.

Each exception should include impact, owner, and next action.

## Evidence Required

A completed BE-10.02 return must include:

```text
Commit List
Files Changed
Deployment Inputs
Configuration Checklist Result
Migration Risk Checklist Result
Rollout Checklist Result
Post-Deploy Verification Plan
Known Exceptions
Deployment Result
Next Step
```

## Handoff

If BE-10.02 passes or deployment is not required, continue to:

```text
BE-10.03 Smoke Verification
```
