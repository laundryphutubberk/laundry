Controllers were reduced or preserved as HTTP request/response orchestration where modified.

Repositories were not changed during the main normalization milestones except where inspected as data-access boundaries. No business decision was intentionally moved into repository files.

## Responsibility Boundary Confirmed

```text
Controller
  owns HTTP request/response orchestration only

Service
  owns business workflow and domain orchestration

Policy / Guard
  owns reusable business assertions, lifecycle guards, and transition eligibility

Repository
  owns data access only
```

## Module Freeze Notes

### Auth

Auth service owns login, registration, token creation, and session assembly.

Auth policy owns login user selection, user active checks, and active membership selection.

### Issue

Issue service owns organization-scoped issue workflow.

Issue policy owns organization context and issue existence assertions.

### Invite

Invite service owns invite workflow and join transaction orchestration.

Invite policy owns invite context, availability, pending, expiration, organization, and user availability assertions.

### Member / Organization

Member and Organization services own membership and organization workflow orchestration.

Their policy files own context and existence assertions.

### Equipment

Equipment service owns equipment runtime CRUD and guarded manual status updates.

Equipment policy owns organization context, equipment existence, and manual status update gate.

Operational checkout/return remains owned by Field Session scan workflow.

### Field Session

Field Session service owns lifecycle orchestration, scan workflow, closure workflow, and movement creation.

Field Session policy owns context, id, existence, and editability assertions.

Field Session lifecycle guard/policy owns status transition and close readiness rules.

### Verification

No active backend verification module is mounted in the current branch.

BE-05.08 records the intended future integration boundary and does not introduce route/API surface.

### Job Finance

No active backend job finance module is mounted in the current branch.

BE-05.09 records the intended future workflow boundary and does not introduce route/API/schema surface.
