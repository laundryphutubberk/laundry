```text
  -> requestContext.middleware
  -> AsyncLocalStorage request context
  -> requestObservabilityMiddleware
  -> structured request log + request counters + latency histogram
  -> route/controller/service
  -> business event logs where implemented

  -> database health check
  -> database timing metric
  -> in-memory metrics snapshot
```

## Request Context Map

```text
Request ID Source:
- inbound x-request-id
- generated crypto.randomUUID fallback

Correlation ID Source:
- inbound x-correlation-id
- fallback to requestId

Propagation:
- req.requestId
- req.correlationId
- response headers x-request-id and x-correlation-id
- req.context
- AsyncLocalStorage via runWithRequestContext
- response meta via httpResponse.buildMeta
- structured logs via observability logger

Context Lifetime:
- starts after requestId middleware
- active during request pipeline via AsyncLocalStorage
- consumed by logger and httpResponse helpers
```

## Risk Assessment

```text
Risk: In-memory metrics reset on restart.
Impact: Production historical metrics are unavailable without external sink.
Recommendation: BE-10 or DevOps phase should define metrics export strategy.

Risk: Audit logging channel is console-based, not persisted.
Impact: Audit events may not satisfy operational or compliance needs.
Recommendation: Create dedicated audit service before production compliance claims.

Risk: No transaction tracing.
Impact: Multi-write workflow diagnosis remains limited.
Recommendation: Add transaction wrapper instrumentation in BE-08/BE-09 follow-up.

Risk: Business event coverage is partial for movement/inventory.
Impact: Inventory lifecycle trace is incomplete.
Recommendation: Add logs when movement and inventory workflows are finalized.
```
