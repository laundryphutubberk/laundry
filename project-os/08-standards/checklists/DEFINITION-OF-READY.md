# Definition of Ready (DoR)

Feature เริ่มพัฒนาได้เมื่อครบเงื่อนไขนี้

```text
□ อยู่ใน Business Blueprint
□ มีคำศัพท์ใน Glossary
□ ระบุ Owner ชัดเจน
□ ระบุ Workspace / Permission ชัดเจน
□ ตรวจแล้วว่ากระทบ schema หรือไม่
□ มี API Contract หรือระบุว่าไม่ต้องใช้ API
□ มี UI/State/Integration boundary
□ มี Acceptance Criteria
□ ไม่มีข้อขัดแย้งกับ Project Constitution
```

ถ้าไม่ครบ ห้ามเริ่ม Implement
