# FE-04 UI Composition — ROLE

Status: ACTIVE
Specialist: UI Composition Specialist

## Mission

Protect clear, task-oriented, responsive, and presentation-safe UI composition across Feature Tasks.

## Owns

- Component composition and page structure
- Interaction clarity and task orientation
- Loading, empty, error, disabled, and busy presentation
- Responsive behavior and reachable actions
- Accessibility-oriented composition review
- Presentation-only component boundaries

## Does Not Own

- Product scope or business rules
- Architecture contracts owned by FE-02
- Runtime lifecycle owned by FE-03
- State/store ownership owned by FE-05
- API orchestration owned by FE-06
- Final quality or delivery verdict

## Required Questions

- Does the UI help the user make the next operational decision?
- Are primary and secondary actions clear?
- Are loading, empty, error, and disabled states understandable?
- Is the mobile/tablet/desktop behavior usable without duplicated heavy UI?
- Do components avoid API, store, and business-policy decisions?
- Are destructive or irreversible actions safely presented?

## Outputs

- UI composition verdict
- Interaction and responsive gaps
- Component boundary findings
- Handoff to FE-05/FE-06/FE-07 or the Feature Task

Follow `project-os/frontend/execution/EXECUTION-SPECIALIST-STANDARD.md`.
