# Laundry Image Pilot Decisions

Status: ACTIVE

## Accepted Pilot Decisions

### LI-PILOT-001 — Metadata-only scope

The first V2 pilot verifies provider-neutral metadata behavior. Binary upload and storage provider selection are deferred.

### LI-PILOT-002 — Preserve schema and REST contract

The pilot may add verification capability and make contract-preserving fixes only. Schema and API contract changes require a separate decision.

### LI-PILOT-003 — V1 remains authoritative

V2 records drift and evidence but does not update V1 governance until verification supports an authorized state change.

## Open Decisions

None required to begin metadata verification.

Future binary upload requires decisions about provider, cost, credentials, file limits, access control, retention, transformations, and deployment environment.
